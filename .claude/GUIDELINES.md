# 코드 작성 가이드라인

## 코드 구조 및 컴포넌트 설계

### 파일 길이 제한
- **각 파일은 500줄 내외로 유지**
- 500줄을 초과하는 경우 컴포넌트 분리를 고려
- 단일 책임 원칙(SRP)을 따라 파일 분할

### 컴포넌트 재사용성
- 재사용 가능한 컴포넌트로 설계
- Props를 통한 유연한 커스터마이징 지원
- 비즈니스 로직과 UI 로직 분리
- 공통 기능은 별도 유틸리티/훅으로 추출

### 파일 분리 기준
다음 상황에서 파일 분리 고려:
- 컴포넌트가 너무 많은 책임을 가질 때
- 독립적으로 테스트 가능한 로직이 있을 때
- 여러 곳에서 재사용될 가능성이 있을 때
- 복잡한 상태 관리가 필요할 때

## 예시

### 나쁜 예
```typescript
// UserDashboard.tsx (800줄)
// - 사용자 정보 표시
// - 차트 렌더링
// - 데이터 페칭
// - 폼 처리
// - 모달 관리
```

### 좋은 예
```typescript
// UserDashboard.tsx (150줄) - 메인 컴포넌트
// UserProfile.tsx (100줄) - 프로필 표시
// UserChart.tsx (120줄) - 차트 컴포넌트
// useUserData.ts (80줄) - 데이터 페칭 훅
// UserForm.tsx (100줄) - 폼 컴포넌트
```

## 적용 시점
- 새로운 기능 개발 시
- 리팩토링 작업 시
- 코드 리뷰 시
